# OpenClaw 是什么？我需要准备啥？

读完这一节，你会用**一句话**说清 OpenClaw 能干什么，并且知道自己在动手之前需要准备哪些东西、有没有缺的。不涉及任何安装步骤，只帮你「心里有数」。

---

## 用一句话说：OpenClaw 能干什么？

**OpenClaw 是一个你自己部署的「桥」：把你常用的聊天软件（飞书、Telegram、WhatsApp 等）和你自己的 AI 助手连在一起。**

你在飞书或 Telegram 里发一句消息，消息会先到这台「桥」上，再由它转给你的 AI 助手；助手回复后，同样经过这座「桥」回到你的聊天窗口。这样一来，你不用每次都打开网页或某个 App，在平时聊天的软件里就能和助手对话。这座「桥」在官方文档里叫 **Gateway**（网关）：你可以把它想成你家门口的「总机」——外面的人（飞书、Telegram）要找你家的 AI，都得先经过它。

OpenClaw 是**开源**的，运行在**你自己的电脑或服务器**上，数据和对话由你把控，不依赖某一家公司的托管服务。

---

## 我需要准备什么？

在动手安装之前，下面几样东西能帮你少走弯路。

### 1. 一台能长期开着的电脑或服务器

OpenClaw 的 **Gateway** 需要一直运行，才能随时接收飞书、Telegram、WhatsApp 等发来的消息。所以你需要：

- 一台**长时间开机**的电脑（比如家里的台式机、笔记本），或者  
- 一台**云服务器 / VPS**（如阿里云、腾讯云、海外 VPS 等）。

如果你只是先在浏览器里和助手聊聊天、暂时不接任何聊天软件，本机临时开一会儿就够；等要「接到飞书 / Telegram」时，再考虑长期运行的环境即可。

### 2. Node 22 或更高版本（必须）

OpenClaw 依赖 **Node**。Node 是什么？可以理解成一种让 JavaScript 在电脑上运行的环境，很多现代工具和网站都靠它。官方要求 **Node 22 或更新版本**。

- **如果你不确定自己有没有装 Node**：打开终端（或「命令提示符」），输入 `node --version`，回车。如果显示 `v22.x.x` 或更大的数字，就满足要求；如果提示「找不到命令」或版本很小，就需要先安装或升级。
- **如果还没有 Node**：不用紧张，下一节会用到的一键安装脚本可以**顺带帮你装好 Node**（在 macOS / Linux 上）；你也可以先按 [官方文档 - Node](https://docs.openclaw.ai/install/node) 自己装。

支持的操作系统：**macOS、Linux、Windows**。在 Windows 上，官方建议用 **WSL2**（Windows 自带的 Linux 子系统）来跑 OpenClaw，体验更稳定。

### 3. 飞书 / Telegram / WhatsApp 账号（可选，想接哪个就准备哪个）

如果你希望**在飞书、Telegram 或 WhatsApp 里**和助手聊天，就需要对应平台的账号（或企业飞书应用）。本教程面向中国用户，主要会提到这三种渠道：

- **飞书**：需要安装 OpenClaw 的飞书插件，并在飞书开放平台创建应用、拿到 App ID 和 App Secret（后面章节会写）。
- **Telegram**：一个 Telegram 账号即可，后续在 OpenClaw 里用 BotFather 创建的机器人对接。
- **WhatsApp**：一般需要单独一个手机号（或虚拟号）专门给助手用，不建议用你日常私人的主号，以免所有私聊都变成「助手输入」。

你**不必三个都有**：先选一个你最常用的，跟着教程打通一个渠道就够用了。不接任何渠道也可以：只用在浏览器里打开 OpenClaw 的「控制台」和助手对话（下一节就会做到）。

在进入下一节「安装并在浏览器里和助手说上话」之前，可以快速过一遍：

| 项目 | 你有 / 能搞定吗？ |
|------|-------------------|
| 有一台能用的电脑（先本机试也行） | ☐ |

## 本节要点

- **OpenClaw** = 你自己部署的一座「桥」（Gateway），把飞书、Telegram、WhatsApp 等聊天软件和你的 AI 助手连在一起，在聊天软件里就能和助手对话。
- **必须有的**：能长期运行的环境（本机或服务器）、Node 22+。

## 常见问题

**Q：我完全没接触过 Node / 命令行，能跟下来吗？**  
可以。教程会给出需要你复制粘贴的命令，你只要在终端里执行即可；安装脚本会在支持的系统上尽量自动处理好 Node。

**Q：钉钉能用吗？**  
当前官方文档里没有单独写钉钉的对接方式，请以 [官方渠道列表](https://docs.openclaw.ai/channels) 或 [多语言文档站 www.openclawx.cloud](https://www.openclawx.cloud) 为准；若后续支持，会在渠道文档中说明。

**Q：Token、Agent 这些词是啥？**  
后面章节第一次用到时都会用一句话解释。你只要先记住：**Agent** 在这里就是指「一个完整的 AI 助手」，有自己的人设和记忆；**Token** 可以理解成模型读/写你一句话时用到的「字数单位」，按量计费。
